注意
该教程最好在理解Linux基础命令和有解压以及玩机经验的人使用
如果你不知道什么是zip mp4 以及Linux的ls dd命令 我不建议参考该教程 否则会出现难以解决的问题 比如大量残余文件在系统 占用存储空间而不会删除

本教程来自个人实践复盘 不具备相当完备的体系与合理解决方案 仅作为参考

这是一个通过安卓手机(因为底层是Linux内核)上termux这个APP来提取mp4格式内嵌zip的方法
本教程用于在手机端处理 伪媒体文件 常见于网盘下载 资源站分享等 一种用于规避审查和封装手段

第一步 获取原始目标文件
	你可能会在百度网盘下载目标文件为mp4 下载时应当选择原画 保证内部文件完整
第二步 准备环境
	在termux里面更新软件包和准备必要的工具
	pkg update -y && pkg upgrade -y
	pkg install -y zip p7zip binutils
	最好依次输入上方指令 回车等待提示指示符$再进行下一步
第三步 侦测目标文件是否含有zip魔数
	# 进入工作目录（请替换为你的实际路径）
	cd ~/storage/downloads

	# 搜索 PK 特征（显示前 5 个匹配项）(这是快速搜索 可能什么也没有 建议用下面的)
	strings "目标文件.mp4" | grep -m 5 "PK"

	# 或者使用更底层的十六进制搜索（推荐）
	grep -oba -P '\x50\x4b\x03\x04' "目标文件.mp4" | head -n 1
	
	#若grep长时间没有反应 可以点击Ctrl➕C退出 运行下面这条指令
        grep -oba "PK" 文件名 | head -n 1

	该步骤目的是查看目标文件是否有zip被隐写
	若输出(示例)
	243764:PK
	2434646:PK
	等有PK文件头标识 则代表该文件确实隐写了zip格式
第四步 获取源数据
	# 列出详细信息，第一列是权限，第五列是精确的字节数
	ls -l "目标文件.mp4"
	确保使用ls命令和-l后缀 不应该是-lh后缀 我们需要字节来做判断 而不是-lh输出的gb等
        记下第五列的数字 比如 24379418164 这是目标文件的总大小 以字节表示
第五步 寻找zip头文件偏移量
	# 搜索 ZIP Local File Header 的十六进制特征
	grep -oba -P '\x50\x4b\x03\x04' "目标文件.mp4" | head -n 1
	如果输出  2013461:PK(示例) 则这个数字就是zip的起始头文件位置
第六步 数据切割
	# 语法：dd if=源文件 of=输出文件 bs=1 skip=偏移量
	# 示例：使用上一步得到的偏移量
	dd if="目标文件.mp4" of="提取文件.zip" bs=1 skip=2013461(示例 需要替换成实际第四步输出数字)
	代表通过dd指令从zip文件头所在字节偏移量处切割
	bs=1代表一个字节一个字节切割 精度最高 防止出错
	但速度很慢 一个2GB的完整zip可能要10-15分钟 需要规划时间(这取决于手机IO瓶颈 没办法的)
第七步 完成提取 
	获得完整zip文件 此时应该和这个mp4在同一个文件夹  
      1	通过安卓解压APP 比如 ZArchiver Pro(推荐 我用的这个)找到目标文件夹和zip文件
	或许这个zip里面是个jpg格式图片 但是可以通过APP解压了
	长按jpg 重命名后缀为zip
	之后单击目标zip 解压到当前文件夹或指定路径 输入密码或没有密码直接解压
	解压成功获得目标文件
      2 通过termux内解压	
	# 1. 强制修复 ZIP 结构
	zip -FF "提取文件.zip" --out "修复后的文件.zip"

	# 2. 使用 7z 解压（支持密码）
	7z x "修复后的文件.zip"
	解压后文件就在当前目录 
	如果你熟悉mv指令或者挂载了内部存储 可以在mt管理器将他转移到你选择的任意目录
